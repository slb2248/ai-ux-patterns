# First Success Flow

**Category:** [Onboarding](https://aiuxplayground.com/patterns/onboarding)  
**Interactive demo:** [https://aiuxplayground.com/pattern/first-success-flow](https://aiuxplayground.com/pattern/first-success-flow)

> Guaranteed first successful AI interaction

## What it is

First success flow is an AI onboarding UX pattern that engineers a near-guaranteed positive first generation (curated prompt, constrained task, or sample context) so new users taste value before facing a blank composer or failure.

## When to use

Critical for AI applications, onboarding flows, and platforms where ensuring a positive first AI interaction determines user adoption and continued usage.

## When not to use

- Returning users who should land on their workspace, not a repeat tutorial.
- Open-ended pro tools where a scripted “success” feels condescending.
- When you cannot actually guarantee success and the scripted path often fails.

## Anti-patterns

- A demo that only works offline and breaks when users try the same prompt live.
- Blocking Skip so experts cannot escape the scripted path.
- Measuring “success” as page view instead of a completed generation the user values.
- Handing users off to a blank chat immediately after claiming success.

## How products use it

| Product | Implementation |
|---------|----------------|
| ChatGPT | Starter prompts and example tasks designed to complete cleanly on first try. |
| Notion AI | Guided first AI actions on a sample page or block. |
| Image generators | Curated first prompts that reliably produce a strong sample image. |
| Lovable / builders | First project paths that ship a visible app quickly. |
