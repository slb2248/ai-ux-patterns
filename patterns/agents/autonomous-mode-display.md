# Autonomous Mode Display

**Category:** [Agents](https://aiuxplayground.com/patterns/agents)  
**Interactive demo:** [https://aiuxplayground.com/pattern/autonomous-mode-display](https://aiuxplayground.com/pattern/autonomous-mode-display)

> Persistent signal when agent runs unattended

## What it is

Autonomous mode display is an AI interface design pattern that surfaces a persistent, unmistakable indicator (a color shift, a badge, a shape, a full-app chrome change) whenever an agent is acting without asking. This UX pattern makes autonomy legible at a glance so users never discover it by accident, which is the single most common failure mode in early agentic products. The indicator has to be durable, not ephemeral: a momentary toast or a small icon change is not enough when the state may persist for minutes or hours. Well-executed mode displays borrow from physical analogs like a microphone-on light or a recording LED, where presence, absence, and state change are obvious from across the room. The pattern is how agents earn the right to run unattended, by making the fact that they are running unattended impossible to miss.

## When to use

Essential for AI coding agents, always-on assistants, and automated workflows where unattended execution needs a glanceable, always-visible signal so users never discover autonomy by surprise.

## Seen in

- Cursor Agent badge

- macOS microphone indicator
- iOS recording indicator
