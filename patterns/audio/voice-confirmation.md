# Voice Confirmation

**Category:** [Audio](https://aiuxplayground.com/patterns/audio)  
**Demo:** [aiuxplayground.com/pattern/voice-confirmation](https://aiuxplayground.com/pattern/voice-confirmation)

> Spoken approval for high-stakes voice actions

## Overview

Voice confirmation is an AI interface design pattern that requires an explicit spoken or multimodal confirmation before a voice agent commits to an irreversible or high-stakes action, rather than executing on a single cheerful 'sure, done that.' This UX pattern inserts a deliberate, legible beat between intent and execution for operations that cannot be cleanly undone (payments, purchases, destructive commands, bulk messages) so users hear or see exactly what is about to happen in the same channel they used to ask. Confirmation can be reinforced with voice match, a visual echo on a paired screen, or a physical button press, calibrated to the risk of the action. The pattern is the voice-native analog of a confirmation modal: a small friction in exchange for a large reduction in the cost of being misheard.

## Good for

Essential for voice shopping, voice banking, smart-home automation, and any always-on agent where irreversible actions need an explicit spoken or multimodal confirmation before execution.

## Seen in

- Alexa voice-match purchasing
- Siri banking confirmation
- Google Assistant voice payments
- Apple Pay voice re-auth
